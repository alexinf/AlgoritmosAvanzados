Llamada rapida

Los contactos de tu teléfono en su mayoria tienen la misma cantidad de cifras, pero hay algunos numeros que son especiales como los de emergencia o servicios especiales o incluso de otros paises que hace que la longitud de los numero no sea igual. 

Usualmnete cuando se usa estos numeros especiales, una de las cosas que se desea es que al escribir el numero se haga la llamada de manera inmediata sin la presion de alguna otra tecla. Para ello la lista de tus contactos no deberia interferir para activar esta funcionalidad.

Cuando un numero de contacto interfiere con esta funcionalidad? cuando el numero especial es prefijo exacto y completo de algun numero de tus contactos.

La tarea es averiguar si se puede usar de manera no ambigua el numero especial o no, en caso de que no se pueda debes indicar qué contactos impiden esta funcionalidad de llamado rapido. Por otro lado la lista de tus contactos estan organizados por la importancia que ellos significan para ti, por lo que no es correcto cambiar el orden de los contactos.

Por ejemplo si se tiene la siguiente lista de contactos:

Lina   42394524
Gaston 42387465
Daura  10454637
Laura  27364525
Jaime  10463524
Juan   43526473

Lina vendria a ser el contacto mas importante ya que esta en primer lugar de la lista.

Cuando alguno(s) de tus contacto(s) interfiere te interesa saber cual es la posicion  de importancia de estos contactos.

Considerando la lista de contactos del ejemplo y si el numero especial al cual se le quiere asignar la llamada rapida es el 104, los contactos 3 y 5 no permiten activar esta funcionalidad. Pero si el numero especial al que se quiere asignar la funcionalidad de llamada rapida es el 123, no hay contactos que impidan la activacion de esta funcionalidad, por lo que la respuesta debe ser "Activado".

Los nombres de los contactos son nombres de "pila'" y tienen a lo sumo 6 caracteres.

El reporte de las posiciones de los contactos a su vez deben mostrarse considerando el orden ascendente de los numeros telefonicos.

Considerando la lista ejemplo si el numero especial es 423, los contactos que correponden a

Gaston 42387465

son los que interfieren y a su vez son los contactos 1 y 2 en importancia, pero el reporte debe entregar la secuencia 2, 1; ya que el numero de telefono del contacto de la 2da posicion es menor que el de la primera.

Le siguen n lineas con dos datos en cada linea: c que es el nombre del contacto  y  t que es el numero del telefono que tiene entre 4 y 20 digitos. El orden en el que se encuentran estos datos corresponde a la importancia del contacto.

La siguiente linea tiene el numero especial ne al cual se quiere activar el llamado rapido. ne puede tener entre 2 y 20 digitos

#!/usr/bin/python
# coding: utf-8

class Node( object ):
	def __init__( self, end_node = False ):
		self.end_node = end_node
		self.prefix_count = 0
		self.children = {}
	


class Trie( object ):
	def __init__( self ):
		self.root = Node()
	
	def insert( self, key ):
		current = self.root
		for k in key:
			if k not in current.children:
				current.children[k] = Node()
			current = current.children[k]
			current.prefix_count += 1
		current.end_node = True

	def enumerate( self, key ):
		current = self.root
		for k in key:
			if k not in current.children:
				return []
			current = current.children[k]
		
		return self._walk( current, key )	

	def _walk( self, root, prefix ):
		out = []
		if root.end_node:
			out.append( prefix )
		
		for ch in root.children:
			if isinstance( prefix, tuple ):
				tmp = self._walk( root.children[ch], prefix + (ch,) )
			elif isinstance( prefix, list ):
				tmp = self._walk( root.children[ch], prefix + [ch] )
			else:
				tmp = self._walk( root.children[ch], prefix + ch )
			out.extend( tmp )
		return out

Cadenas equilibradas

En el pais de los "buenos", todas las cadenas se escriben considerando el alfabeto reducido {b,n,s,u,e,o}, en este alfabeto las consonantes tienen menor valor que las vocales y el orden lexicografico de este alfabeto es: b < n < s < u < e < o; el valor numerico asociado a cada letra del alfabeto reducido es 
0, 1, 2, 3, 4, 5 respectivamente.

Las cadenas tienen un peso asociado, que es la suma de los valores de los caracteres que tiene, por ejemplo "unos" tiene un peso total de 11 = 3 + 1 + 5 + 2. Esta cadena esta desequilibrada ya que no se puede distribuir su peso de forma equitativa en las mitades de la cadena, esto porque el peso es impar y no se puede dividir de forma exacta.

Pero si la cadena es "unes" el peso total es 10 = 3+1+4+2; pero aunque su peso sea par esta cadena no es equilibrada ya que no se puede distribuir de manera equitativa el peso en las mitades extremas  de la cadena, ya que si se divide la cadena en dos partes iguales de caracteres (siempre los n/2 primeros caracteres y los n/2 ultimos caracteres) se tendra por un lado "un" y por otro "es" y el peso de estas partes no son iguales.

La tarea en esta ocasión es decidir cuantas cadenas equilibradas de peso total p y cantidad de n caracteres se pueden formar con este alfabeto reducido.

Por ejemplo, si te dan como dato que el peso es 10 y el largo de la cadena es 4, entonces se pueden formar 36 cadenas diferentes y equilibradas, entre ellas se tiene "bobo", "boob"......"usus".

Como podras imaginar, las cadenas que tienen cantidad de caracteres impar, no pueden ser equilibradas ya que no se puede repartir el peso por igual.

Entrada

Se tienen varios casos de prueba que terminan con el fin de archivo. 
Para cada caso se dan dos numeros: n que es la cantidad de caracteres que deben tener las cadenas y p que es el peso total que se quiere distribuir.

Puedes asumir que 0 ≤ p ≤ 1000 y 1 ≤ n ≤ 100.

Salida

Por cada caso se debe imprimir en una linea la cantidad de cadenas equilibradas de longitud n y peso p que pueden haber con el alfabeto reducido.

Sample Input
4 4
4 10
2 12
3 10
4 5

36
